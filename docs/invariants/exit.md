# Exit / Portability Invariants (EXIT) — Draft

## Scope
- Applies to portability, exit planning, validation, and irreversibility compensation.
- Covers declared exit paths and executed validation.

## Invariants
- **EXIT-01 — Exit Path Declaration**  
  Exit paths MUST be declared for governed services.
- **EXIT-02 — Bounded Exit Feasibility**  
  Exit MUST be feasible within defined bounds (time, data volume, cost).
- **EXIT-03 — Executed Exit Validation**  
  Exit procedures MUST be exercised and evidenced.
- **EXIT-04 — Irreversibility Compensation**  
  Irreversible steps MUST be identified and compensated.

## Evidence expectations
- Exit plans, validation exercises, and results.
- Export/import evidence and integrity checks.
- Compensation measures for irreversibility.

## Non-goals
- No mandated commercial terms or pricing structures.
- No requirement to support specific migration tools.
